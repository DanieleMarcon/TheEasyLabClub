# 08-seo-geo-aeo.md — The EasyLab Club

## 1. Purpose

Define rules for:

* SEO (Search Engines)
* GEO (Generative Engines)
* AEO (Answer Engines)

This ensures visibility across:

* Google
* AI search systems
* LLM-based interfaces

---

## 2. Core Principle

Content must be:

* clear
* structured
* entity-based

Not:

* keyword-stuffed
* vague
* overly optimized

---

## 3. SEO Rules (Search Engines)

### URL Structure

* short
* readable
* semantic

Example:

* /programs
* /lab
* /lab/build-ai-workflows

---

### Headings

* one H1 per page
* logical hierarchy (H1 → H2 → H3)
* no skipping levels

---

### Metadata

Each page must include:

* title (clear and descriptive)
* meta description
* open graph tags

---

### Internal Linking

* link between Programs, Lab, Network
* avoid orphan pages

---

## 4. GEO Rules (Generative Engines)

### Entity Clarity

Each page must clearly define:

* what it is
* who it is for
* what problem it solves

---

### Structured Content

Use:

* short paragraphs
* lists
* clear sections

---

### Explicit Meaning

Avoid:

* implicit assumptions
* hidden context

Content must be self-contained.

---

## 5. AEO Rules (Answer Engines)

### Question-Based Sections

Include:

* short Q&A blocks
* clear answers

---

### Direct Answers

* first sentence must answer the question
* no long introductions

---

### FAQ Sections

Use for:

* Programs
* Lab
* General pages

---

## 6. Content Structure Rules

Each page must:

* have a clear topic
* avoid mixing multiple intents
* stay focused

---

## 7. Naming Rules

Use consistent naming:

* Programs
* Lab Modules
* Network

No variations allowed.

---

## 8. Performance (SEO Impact)

* fast loading
* optimized images
* minimal scripts

---

## 9. Multi-language (Future)

Prepare for:

* /en
* /it

Use:

* consistent structure
* hreflang tags (future)

---

## 10. Critical Rule

Clarity improves:

* SEO
* GEO
* AEO

If content is unclear:
→ it will not rank
→ it will not be retrieved
→ it will not be answered
