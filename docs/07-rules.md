# 07-rules.md — The EasyLab Club

## 1. Purpose

Define global rules for:

* UX
* content
* structure
* AI behavior

These rules prevent inconsistency and design drift.

---

## 2. Core UX Rules

### Clarity First

* Users must understand the offer immediately
* No hidden meaning
* No abstract messaging

---

### One Goal per Section

Each section must have:

* one clear purpose
* one clear message

Avoid:

* mixed messages
* overloaded sections

---

### Strong Hierarchy

* clear visual order
* clear reading flow
* no ambiguity

---

## 3. Navigation Rules

Allowed navigation:

* Home
* Programs
* Lab
* Network
* About
* Login

Rules:

* no additional primary items
* no duplicates (e.g. no “For Teams”)
* no conceptual navigation (Learn / Apply / Improve)

---

## 4. Programs vs Lab (Critical)

This distinction must always be clear.

### Programs

* for teams
* structured paths
* consultation-based
* no direct purchase

### Lab

* for individuals
* modular
* direct purchase
* fixed pricing

Never mix:

* pricing models
* messaging
* CTAs

---

## 5. CTA Rules

### Programs

* Request a consultation
* Build your Program

### Lab

* Start here
* Access the modules

Rules:

* CTA must match context
* no generic CTAs
* max 2 per section

---

## 6. Content Rules

### Language

* simple
* direct
* concrete

Avoid:

* jargon
* buzzwords
* abstract concepts

---

### Tone

* professional but not corporate
* modern but not trendy
* clear but not simplistic

---

### Copy Length

* short paragraphs
* no long blocks of text
* no unnecessary explanation

---

## 7. Design Behavior Rules

### No Visual Noise

* no unnecessary elements
* no decorative overload

---

### Consistency

* same components reused
* same spacing logic
* same typography rules

---

### No Experimental Layouts

* no complex grids
* no overlapping sections
* no unpredictable UI

---

## 8. Network Rules

Network is:

* secondary
* aspirational
* not a community platform

Avoid:

* social features
* community UI patterns
* excessive detail

---

## 9. AI-Specific Rules

### No Interpretation

AI must:

* follow documentation
* not invent features
* not change structure

---

### No Creativity Drift

AI must not:

* change tone
* introduce new concepts
* create new sections

---

### Strict Adherence

If something is not defined:
→ do not invent it

---

## 10. Error Prevention Rules

Avoid:

* mixing Programs and Lab
* adding unnecessary pages
* changing navigation
* altering CTA logic

---

## 11. Final Rule

If something reduces clarity:
→ it must be removed

Clarity is always the priority.
