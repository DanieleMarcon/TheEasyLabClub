# 06-components.md — The EasyLab Club

## 1. Purpose

Define all reusable UI components.

Each component must be:

* modular
* predictable
* reusable

This document is used directly by AI to generate UI.

---

## 2. Page Structure (Global Pattern)

All pages follow this structure:

1. Hero
2. Intro / Context
3. Core Content (varies by page)
4. Supporting Sections
5. CTA
6. Footer

---

## 3. HERO COMPONENT

### Structure

* headline
* subheadline
* primary CTA
* secondary CTA (optional)

### Rules

* must communicate value immediately
* no abstract messaging
* no long paragraphs

---

## 4. PROBLEM SECTION

### Purpose

Explain the core issue clearly.

### Structure

* short headline
* short paragraph

### Rules

* simple language
* no jargon

---

## 5. SOLUTION SECTION

### Purpose

Explain how EasyLab works.

### Structure

* short explanation
* optional steps

### Example pattern

* Learn
* Apply
* Improve

---

## 6. SPLIT SECTION (CRITICAL)

Used to separate:

* Programs
* Lab

### Structure

Two blocks side by side:

LEFT → Programs
RIGHT → Lab

Each contains:

* title
* short description
* CTA

### Rules

* must be visually distinct
* must be immediately understandable

---

## 7. PROGRAM CARD

Used in Programs page.

### Structure

* title
* short outcome description

### Rules

* focus on results
* no technical jargon

---

## 8. PROCESS SECTION (Programs)

### Purpose

Explain how Programs work.

### Structure

Steps:

1. analysis
2. goal definition
3. structured program
4. application
5. optimization

### Rules

* sequential
* clear progression

---

## 9. OUTCOME SECTION

### Purpose

Show benefits.

### Structure

List of outcomes:

* reduced time
* clearer processes
* better use of tools
* increased autonomy

### Rules

* concrete
* no vague claims

---

## 10. LAB MODULE CARD

### Structure

* title
* short description
* price

### Rules

* price must be clearly visible
* simple layout
* no clutter

---

## 11. FREE ENTRY SECTION

### Purpose

Introduce free content.

### Structure

* short intro
* list of items

### Rules

* positioned before premium modules
* acts as onboarding

---

## 12. NETWORK SECTION

### Purpose

Introduce continuity layer.

### Structure

* short headline
* short explanation

### Rules

* minimal
* no feature overload
* no community-style layout

---

## 13. CTA BLOCK

### Purpose

Drive action.

### Structure

* strong headline
* one or two buttons

### Rules

* clear action
* no confusion

---

## 14. FOOTER

### Structure

* logo
* navigation links
* legal links

### Rules

* simple
* no clutter

---

## 15. COMPONENT RULES

All components must:

* be reusable
* follow same spacing rules
* use same typography system

No:

* one-off designs
* inconsistent layouts

---

## 16. FINAL PRINCIPLE

UI is built from components.

Pages are compositions of components.

No component = no UI.
