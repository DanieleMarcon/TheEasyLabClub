# 04-ui-audit.md — UI Audit (Stitch Output)

## Goal

Translate Stitch outputs into actionable UI decisions.

This document defines:

* what to keep
* what to remove
* what to adapt

It prevents design drift before moving into Bolt.

---

## 1. KEEP (Approved Elements)

### Layout Structure (Home v1)

* Hero → Problem → Solution → Split (Programs / Lab) → Network → CTA
* Clear section hierarchy
* Strong vertical flow

### Programs Page Structure

* Hero + CTA (consultation)
* Explanation (not a course)
* Process / methodology
* Outcomes
* Example programs
* Final consultation CTA

### Lab Page Structure

* Free entry section
* Premium modules grid
* Fixed pricing visible
* Modular explanation
* CTA at bottom

### Core UI Patterns

* Section-based layout
* Card-based modules
* Split blocks (Programs vs Lab)
* Clear CTA hierarchy

---

## 2. REMOVE (Not Aligned)

### Branding Drift (Critical)

* "Clinical Atelier"
* "Precision Engine"
* "Operational Architecture" (overused)
* Any luxury/editorial tone

### Navigation Issues

* Learn / Apply / Improve (as primary nav)
* For Teams (duplicate of Programs)
* Curriculum / Community labels

### Tone Problems

* Overly aggressive performance language
* Elite / exclusive tone
* Abstract metaphors replacing clarity

### Network Overexposure

* Member counts (e.g. 2,400+)
* Over-detailed features (vault, chapters, hubs)
* Community-style visuals

---

## 3. ADAPT (Refine)

### Hero Section

FROM:

* manifesto style
* abstract phrasing

TO:

* clear value
* direct benefit
* simple CTA

---

### Programs Section

FROM:

* conceptual / design-heavy

TO:

* outcome-driven
* operational clarity
* business relevance

---

### Lab Section

FROM:

* aesthetic emphasis

TO:

* product clarity
* price visibility
* modular logic

---

### Network Section

FROM:

* feature-heavy

TO:

* aspirational layer
* continuity message

---

## 4. DESIGN DIRECTION (FINAL)

### Visual Direction

* clean
* structured
* modern
* minimal

### Avoid

* startup flashy UI
* luxury editorial look
* over-designed layouts

---

## 5. KEY RULE

UI reference is NOT product truth.

It is:
→ a structural guide
→ not a branding system
→ not final copy

---

## 6. OUTPUT OF THIS STEP

After this audit:

* UI is reduced to structure
* design noise is removed
* system is ready for component definition
