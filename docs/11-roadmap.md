# 11-roadmap.md — The EasyLab Club

## 1. Purpose

Define execution phases and priorities.

This prevents:

* overbuilding
* scope creep
* loss of focus

---

## 2. Core Principle

Build only what is needed for:

→ a functional and clear V1

Everything else is planned, not built.

---

## 3. Phase Structure

The project is divided into phases:

* V1 → Launch
* V2 → Expansion
* V3 → System

---

## 4. V1 — Launch (Current Phase)

### Goal

Build a complete, clear and functional website.

---

### Must include

* Home
* Programs page
* Lab page
* Network page
* About page

---

### Must include (functional)

* clear navigation
* clear CTAs
* structured content
* responsive layout

---

### Must NOT include

* user accounts
* real checkout
* backend system
* complex integrations

---

### Optional (light)

* simple contact form
* placeholder checkout
* basic analytics

---

## 5. V2 — Expansion

### Goal

Enable monetization and user interaction.

---

### Includes

* real checkout (Stripe, PayPal, Revolut)
* user accounts
* basic dashboard
* access control (Lab modules)

---

### Improvements

* content expansion
* better SEO structure
* initial automation

---

## 6. V3 — System

### Goal

Transform into a complete operational platform.

---

### Includes

* full backend
* CRM integration
* advanced automation
* AI integration
* personalized experience

---

## 7. Priorities

Always prioritize:

1. clarity
2. structure
3. usability

Not:

* features
* complexity
* visual perfection

---

## 8. Build Order (V1)

1. Home
2. Programs
3. Lab
4. Network
5. About

---

## 9. Iteration Rules

* build one page at a time
* validate before moving forward
* avoid parallel development

---

## 10. Feature Validation

Before adding anything:

Ask:

* does it improve clarity?
* is it needed for V1?
* does it align with strategy?

If not:
→ do not build

---

## 11. Anti-Scope Creep Rule

If a feature is not in V1:

→ move it to V2 or V3

Do not force it into V1.

---

## 12. Completion Criteria (V1)

V1 is complete when:

* all core pages exist
* navigation is clear
* Programs vs Lab is unmistakable
* CTAs are working
* design is consistent

---

## 13. Final Principle

Execution over perfection.

A working system beats a perfect idea.
