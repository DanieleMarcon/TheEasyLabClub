# The EasyLab Club --- Website Project

## Overview

This repository contains the structure, documentation and assets for
building the EasyLab Club website.

The project follows an AI-driven workflow, using structured
documentation as the source of truth and Bolt.new as the main execution
tool.

------------------------------------------------------------------------

## Goal

Build a website that clearly communicates the EasyLab Club system and
supports:

-   Programs (team transformation)
-   Lab (modular learning)
-   Network (continuity)

------------------------------------------------------------------------

## Architecture

### Docs

All decisions are defined in `/docs`.

Key files: - 00-project-overview.md - 01-strategy.md -
02-site-architecture.md - 03-content.md

These guide AI execution and must stay consistent.

------------------------------------------------------------------------

### Content

Structured content lives in `/content`: - programs - labs - pages

------------------------------------------------------------------------

### References

-   UI → `/references/ui`
-   Copy → `/references/copy`

------------------------------------------------------------------------

### Source

`/src` will contain generated code.

------------------------------------------------------------------------

## Workflow

1.  Define strategy and content
2.  Create UI references (Stitch)
3.  Generate structure (Bolt)
4.  Iterate modularly

------------------------------------------------------------------------

## Principles

-   clarity over complexity
-   real content over placeholders
-   modular structure
-   minimal token usage
-   documentation-driven development
